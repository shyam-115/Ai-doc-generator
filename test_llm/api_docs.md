# Ai-doc-generator API Documentation

This document provides comprehensive API documentation for the Ai-doc-generator service, detailing its purpose, authentication methods, available endpoints, and common error responses.

## 1. Overview

The Ai-doc-generator API provides an AI-powered service to automatically generate documentation from various sources (e.g., code repositories, URLs). It supports asynchronous job processing for complex generation tasks and offers endpoints to manage and retrieve generated documentation.

*   **Purpose:** Automate the generation of technical documentation using AI.
*   **Base URL:** `https://api.aidocgenerator.com/v1` (This is a placeholder; replace with your actual API base URL)
*   **Authentication:** Bearer Token (JWT)

## 2. Authentication

The Ai-doc-generator API uses Bearer Token authentication. A valid JSON Web Token (JWT) must be included in the `Authorization` header of every request.

**How to authenticate:**

1.  Obtain a valid JWT (e.g., by logging in through a separate authentication service or an admin panel).
2.  Include the token in the `Authorization` header of your HTTP requests, prefixed with `Bearer `.

**Example Header:**

```
Authorization: Bearer YOUR_JWT_TOKEN_HERE
```

Failure to provide a valid token will result in a `401 Unauthorized` or `403 Forbidden` response.

## 3. Endpoints

### 3.1. `POST /generate`

Initiates an asynchronous AI-powered documentation generation job.

*   **Description:** This endpoint accepts parameters defining the source of the documentation (e.g., a repository URL, a specific file URL) and desired output format. It returns a job ID which can be used to track the generation process.
*   **Handler:** `generate_docs` (fastapi)

#### Request

*   **Method:** `POST`
*   **Path:** `/generate`
*   **Content-Type:** `application/json`
