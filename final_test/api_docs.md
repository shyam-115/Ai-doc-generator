# Ai-doc-generator API Documentation

This document provides comprehensive API documentation for the "Ai-doc-generator" service, designed to help developers integrate with its functionalities for automated documentation generation.

## 1. Overview

The Ai-doc-generator API allows you to programmatically generate documentation from various sources, manage generation jobs, and monitor the service's health. It supports asynchronous documentation generation, providing flexibility for complex tasks.

*   **Purpose**: Automate the generation of technical documentation from codebases or other structured inputs.
*   **Base URL**: `https://api.ai-doc-generator.com/v1` (Note: This is a placeholder. Please replace with the actual base URL of your deployment.)
*   **Authentication**: All protected endpoints require authentication using a Bearer Token.

## 2. Authentication

The Ai-doc-generator API uses **Bearer Token** authentication. To access protected endpoints, you must include an `Authorization` header with your request, containing a valid JWT (JSON Web Token).

**Header Example:**

```
Authorization: Bearer YOUR_ACCESS_TOKEN
```

Replace `YOUR_ACCESS_TOKEN` with the actual token obtained after successful authentication (e.g., from a login endpoint not covered in this documentation, or an API key management system).

## 3. Endpoints

### `POST /generate`

Generates documentation asynchronously based on the provided input.

*   **Description**: Initiates a documentation generation job. The API will process the input and return a `job_id` which can be used to query the status and result of the generation.
*   **Handler**: `generate_docs` (main.py, FastAPI)

#### Request Parameters

None (all input is via request body).

#### Request Body Schema

The request body should be a JSON object specifying the source content and desired generation parameters.
